### Sandbox and Admin

**New Features**

*   SandboxManager: Proxy `get_status` through the sandbox proxy API and initialize the state machine lazily only when a transition is needed ([#1213](https://github.com/alibaba/ROCK/pull/1213))

*   Scheduler: Add a Docker health task to automatically restart exited Docker daemons on workers ([#1190](https://github.com/alibaba/ROCK/pull/1190))

**Bug Fixes**

*   Metrics: Adjust request error metrics handling for `BadRequestRockError` ([#1139](https://github.com/alibaba/ROCK/pull/1139))

### Archive

**New Features**

**Bug Fixes**

### Performance and Reliability

**New Features**

*   Logger: Make multi-worker log file appends safe and reset logs at deploy time ([#1195](https://github.com/alibaba/ROCK/pull/1195))

*   Proxy: Reuse shared `httpx` clients instead of creating per-request clients ([#1196](https://github.com/alibaba/ROCK/pull/1196))

*   Admin: Adopt `uvloop`/`httptools` and simplify request metrics ([#1201](https://github.com/alibaba/ROCK/pull/1201))

*   Admin: Reuse a shared `httpx` client for registry manifest probes ([#1205](https://github.com/alibaba/ROCK/pull/1205))

*   Admin: Add a dedicated Ray executor thread pool and improve error observability ([#1209](https://github.com/alibaba/ROCK/pull/1209))

*   Admin: Move the DB layer to a sync engine with a dedicated thread pool to fix event-loop starvation ([#1211](https://github.com/alibaba/ROCK/pull/1211))
