### Scheduler

**New Features**

*   Switch FileCleanupTask to `find -delete` with minimal path safety guards ([#967](https://github.com/alibaba/ROCK/pull/967))

*   Add DB-driven SandboxLogArchiveTask, replacing legacy sentinel file design ([#1025](https://github.com/alibaba/ROCK/pull/1025))

*   Enhanced Ray log cleanup: ([#1029](https://github.com/alibaba/ROCK/pull/1029))

    *   PART 2c: Clean `runtime_env_setup-*` files (covers hex suffix)

    *   PART 2d: Clean rotated daemon logs (`raylet.N.out`, `gcs_server.N.err`, etc.)

    *   PID-aware cleanup for `session_latest/logs` + `logs/old` directory

    *   Protect `agent-*` and other daemon files from PID probe false positives

*   Deduplicate region scheduler.tasks via base config inheritance ([#1003](https://github.com/alibaba/ROCK/pull/1003))

**Bug Fixes**

*   FileCleanupTask: Fix `exclude_dirs` whitelist ineffective due to `-depth` disabling `-prune`, replaced with `-not -path` ([#1072](https://github.com/alibaba/ROCK/pull/1072))

*   FileCleanupTask: Fix PID/TID reuse false positives in `check_pid_exists`, add process name verification ([#1074](https://github.com/alibaba/ROCK/pull/1074))

*   FileCleanupTask: Use `find -type d` in `_discover_candidates` to skip daemon log files ([#1025](https://github.com/alibaba/ROCK/pull/1025))

*   ImageCleanupTask: Split idempotent prune from docuum launch logic ([#1023](https://github.com/alibaba/ROCK/pull/1023))

*   SandboxLogArchiveTask: Fix cross-event-loop asyncpg pool issue, dispatch DB calls to main loop ([#1025](https://github.com/alibaba/ROCK/pull/1025))

*   Scheduler: Add 60s timeout cap on cross-loop dispatch to prevent hang ([#1025](https://github.com/alibaba/ROCK/pull/1025))
